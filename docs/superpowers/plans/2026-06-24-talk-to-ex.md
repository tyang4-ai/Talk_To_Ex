# Talk_To_Ex Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan epic-by-epic. Steps use checkbox (`- [ ]`) syntax for tracking. Each epic ends with an independently testable deliverable and a commit.

**Goal:** Build the full self-hosted infrastructure for an SMS service where a friend subscribes (Stripe), uploads their ex's chat history, and texts a number to chat with a local-LLM persona of that ex — delivered as a code scaffold with placeholder secrets and mock-based tests, pushed to `tyang4-ai/Talk_To_Ex`, plus a continuation prompt for live bring-up.

**Architecture:** One FastAPI monolith (localhost:8080) serving a polished React SPA + REST API; SQLite via SQLModel; Fernet-encrypted sensitive data; Claude API for one-time persona distillation + periodic style re-tuning; local Ollama/Qwen for live SMS replies; Twilio webhook (empty-TwiML-ack + async REST send); Cloudflare named tunnel for public ingress. See `docs/superpowers/specs/2026-06-24-talk-to-ex-design.md`.

**Tech Stack:** Python 3.11, FastAPI, SQLModel, uvicorn, PyJWT, passlib[bcrypt], cryptography(Fernet), httpx, anthropic, twilio, stripe, pytypedstream, ftfy, pdfminer.six, chat-miner (vendored), pytest + respx. Frontend: Vite + React 18 + TS + Tailwind + framer-motion + @stripe/stripe-js. Ops: Ollama, cloudflared.

## Global Constraints

- **All secrets are placeholders** (`__PLACEHOLDER__`); no live API calls this session. Config loads via a typed settings object; missing secrets fail at the boundary call, never at import — the app must import and the mock test suite must pass with no real keys.
- **Encrypt at rest:** persona/memory/style artifacts + raw+normalized uploads use Fernet (`FERNET_KEY`).
- **Local model never tunnelled:** Ollama stays on `127.0.0.1:11434`.
- **Twilio replies are async:** webhook returns empty `<Response/>` immediately; real reply sent via REST `Messages.create`.
- **Ollama params, always explicit:** `num_ctx:8192`, `keep_alive:"-1"`, env `OLLAMA_NUM_PARALLEL=1`; model `qwen2.5:14b-instruct-q4_K_M`, embed `bge-m3`.
- **Distillation model:** `ANTHROPIC_MODEL` (default `claude-opus-4-8`).
- **Core personality is frozen:** style re-tuning (`STYLE_RETUNE_EVERY=100`) may modify Layer 2 (expression) only; a validation rejects any overlay that changes Layers 0/1/3/4/5.
- **Safety is deterministic and pre-model:** bilingual crisis tripwire runs before the LLM; on hit, bypass persona → static 988+hotline reply + operator alert + `safety_events` log.
- **Mixed zh/en:** parsers and prompts must handle Chinese + English; live system prompt mirrors the user's language.
- **TDD + frequent commits:** test-first per task; conventional-commit messages; one commit per task.

## Epic E1 — Ingestion parsers (parallelizable; one sub-task per format)

**Files:** `app/ingestion/parsers/*`, `app/ingestion/normalize.py`, tests + fixtures `tests/ingestion/fixtures/*`.

**Interfaces:**
- `parsers/base.py`: `@dataclass NormalizedMessage{sender:str, ts:datetime, text:str, direction:Literal["in","out"]}`; `NormalizedTranscript = list[NormalizedMessage]`; `class Parser(Protocol){ def parse(self, path:str, target:str|None=None)->NormalizedTranscript }`.
- `parsers/detect.py`: `detect_format(path:str)->Literal["imessage","instagram","whatsapp","wechat","plaintext"]` (sniff SQLite magic `b"SQLite format 3\\x00"`, JSON keys `messages/sender_name`, `_chat.txt` bracket/dash header, XML `<smses>` root, else plaintext).
- Each parser exports `parse(path, target=None)->NormalizedTranscript`.

**Per-format requirements (the verified gotchas — implement exactly):**
- `instagram.py`: glob `**/messages/inbox/*/message_*.json`; concat `messages[]`; **sort ascending by `timestamp_ms`**; fix each text field with `ftfy.fix_text` (NOT bare latin-1); support old(`sender/text`)+new(`sender_name/content`) keys.
- `whatsapp.py`: vendor `chat-miner` parsing logic or implement its regex set (iOS `[D/M/Y, H:M:S] S:` + Android `M/D/Y, H:M AM - S:`), strip U+200E/BOM, merge continuation lines, drop system/`<Media omitted>` lines; tolerate `上午/下午`.
- `imessage.py`: open SQLite `sms.db`; join `message/handle/chat/chat_message_join`; if `text` NULL decode `attributedBody` via `pytypedstream`; date = per-row magnitude branch (`ns` vs `s`, `+978307200`); `is_from_me` → direction.
- `wechat.py`: best-effort — accept a decrypted/exported `.txt`/`.csv`/`.html` (reuse ex-skill `wechat_parser` regex + media-tag stripping); on encrypted DB input, raise `ParserNeedsManualExport`.
- `sms.py`: Android "SMS Backup & Restore" XML (`<sms address body date type>`, type 1=in/2=out) + CSV.
- `plaintext.py`: pdfminer.six for `.pdf`, else read text; ex-skill regex `^<date> <sender>[:：] <content>`.

- [ ] For EACH parser: test against a fixture asserting count + first/last message text+direction (incl. a Chinese line and the mojibake/attributedBody/locale case) → fails → implement → passes → commit `feat(ingestion): <format> parser`.

## Epic E3 — Conversation engine + style tuner

**Files:** `app/convo/{ollama_client.py,history.py,summary.py,engine.py,style_tuner.py}`, tests.

**Interfaces:**
- `ollama_client.chat(messages, *, num_ctx=8192, temperature=0.8)->str` and `embed(text)->list[float]` (httpx to `settings.ollama_base_url`, `keep_alive:"-1"`). Injected/mockable.
- `history.append(conv, direction, body)`, `history.recent(conv, n=20)->list[Message]`.
- `summary.maybe_resummarize(conv)` every ~25 msgs → updates `conversation.summary`.
- `engine.reply(persona_id, peer_e164, body)->list[str]` (bubbles): loads frozen persona(0,1,3,4,5)+latest style overlay(2)+summary+recent turns+SMS few-shot+language rule → `ollama_client.chat` → split on bubble delimiter `\\n---\\n`.
- `style_tuner.maybe_retune(conv)`: if `conv.message_count % settings.style_retune_every == 0` → build messages from frozen persona + last 100 turns → anthropic → new Layer-2 overlay → **`validate_core_unchanged(original, overlay)`** (raise+skip if any core layer differs) → `persona/store` saves overlay + `StyleTuning` row.

- [ ] Test `engine.reply` with **fake ollama** returning `"hey\n---\nu up?"` → asserts `["hey","u up?"]` and persona/system prompt includes the persona name + language rule → fails → implement → passes → commit.
- [ ] Test `style_tuner` rejects an overlay that mutates Layer 0 (fake anthropic returns a tampered core) and accepts a valid Layer-2-only overlay → commit `feat(convo): style re-tuning with core-freeze guardrail`.

## Epic E4 — Messaging gateway + safety

**Files:** `app/messaging/{crisis_words.py,safety.py,sender.py,twilio_webhook.py}`, tests.

**Interfaces:**
- `crisis_words.py`: bilingual regex/keyword sets (en + zh self-harm/suicide).
- `safety.check(body)->SafetyVerdict{crisis:bool}`; `safety.handle_crisis(conv)` → sends static 988+hotline message via `sender`, logs `SafetyEvent`, alerts operator (log/email stub).
- `sender.send_bubbles(to, from_, bubbles)` — Twilio REST `Messages.create` per bubble, randomized 1–3 s sleep (injectable sleeper for tests). Twilio client injected.
- `twilio_webhook` route `POST /sms`: validate `X-Twilio-Signature` (RequestValidator) → parse `From/Body/To` → kill-switch + rate-cap check → `safety.check`; if crisis → handle + return empty TwiML; else schedule BackgroundTask(`_respond`) and **return empty `<Response/>`**. `_respond` = `engine.reply` → `sender.send_bubbles` → `history` persist → `summary.maybe_resummarize` → `style_tuner.maybe_retune`.

- [ ] Test webhook returns `200` + body `<Response></Response>` and schedules a task; signature-fail → 403 → commit.
- [ ] Test crisis input ("我想自杀"/"i want to kill myself") → bypasses engine, sends hotline, logs SafetyEvent → fails → implement → passes → commit `feat(messaging): twilio webhook + deterministic crisis safety`.
