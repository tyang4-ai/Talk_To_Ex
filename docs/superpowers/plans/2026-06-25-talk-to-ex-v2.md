## Epic E12 — Async job queue + worker (foundation for E13/E14)

**Files:** `app/db.py` (`Job` model, §28), `app/jobs/{queue.py,worker.py}`, tests.

**Interfaces:**
- `Job` table: `(id, persona_id, kind, status[queued|training|ready|failed], adapter_path, error, created_at, updated_at)`.
- `queue.enqueue(persona_id, kind="finetune") -> Job`; `queue.claim_next() -> Job|None`; `queue.mark(job, status, **fields)`.
- `worker.run_once()` claims one queued job, dispatches by kind, updates status; a thin loop (or cron-style poll) drives it out-of-process so a 2-day train never blocks the web app. **Single concurrent training job** (GPU contention, §23.4).

- [ ] Test: enqueue → `claim_next` returns it once (not twice) → `mark(ready)` transitions and is idempotent → commit `feat(jobs): persisted async job queue`.
- [ ] Test: `worker.run_once` with a **fake trainer** (DI) flips a job queued→training→ready and records `adapter_path` → commit.

## Epic E13 — Fine-tuning pipeline (§23) — the heavy epic, runs on the 4090

**Files:** `app/finetune/{dataprep.py,train.py,convert.py,serve.py}`, `ops/finetune/` (env + notes), tests (mock the trainer/subprocess).

**Interfaces (all trainer/subprocess calls injected so tests never touch a GPU):**
- `dataprep.build_examples(transcript) -> list[ChatExample]` — ex (`in`) = assistant, friend (`out`) = user; preserve zh/en; window into short SMS-style multi-turn samples; hold out a few turns for a sanity check.
- `train.qlora(examples, base_model, out_dir, *, trainer="unsloth")` — QLoRA via Unsloth (default) or LLaMA-Factory → safetensors adapter. Caps epochs (overfit guard).
- `convert.to_gguf(adapter_dir) -> gguf_path` — llama.cpp `convert_lora_to_gguf.py`.
- `serve.register_adapter(persona_id, gguf_path, base_model) -> ollama_model_name` — writes a Modelfile (`FROM <base>` + `ADAPTER <gguf>` + persona stub) and `ollama create persona-<id>`; persists `adapter_model` on the persona.
- The fine-tune `Job` kind (E12) chains these; on success sets persona `adapter_model` and flips status `building→ready`.

- [ ] Test `dataprep` shapes a normalized transcript into chat examples with correct role mapping + preserved CJK → commit.
- [ ] Test the pipeline orchestration with a **fake trainer + fake converter + fake ollama** (DI): job runs dataprep→train→convert→register, persona ends with `adapter_model="persona-<id>"` → commit `feat(finetune): QLoRA pipeline (mock-tested)`.
- [ ] **Hardware bring-up (host-only, documented in `ops/finetune/`):** prove one real QLoRA run end-to-end on the 4090, GGUF-adapter loads in Ollama, derived model answers in-character and still language-mirrors (no verbatim parroting). **Verify adapter↔base compatibility day one** (§23.4).
- [ ] Engine: when a persona has `adapter_model`, `engine.reply` targets it instead of the routed base model → test → commit `feat(convo): serve per-persona fine-tuned adapter`.

> If `FINETUNE_ENABLED=false`, the persona falls back to the prompt-only path (E2/E3) — the service still works without the fine-tune.

## Epic E14 — Reveal flow + proactive opener (§24)

**Files:** `app/messaging/{sender.py,opener.py,safety.py}`, `app/persona/routes.py` (status transitions), `app/messaging/crisis_words.py` (reuse), tests.

**Interfaces:**
- Persona status gains `building` (upload→distill→enqueue finetune) and `ready/active` (job done).
- `opener.first_message(persona) -> str` — picks one of 3–4 **templated** in-character apology variants (NOT free generation, §24.2).
- On job `ready`: activate the persona and `sender.send_bubbles(...)` the opener **after** an outbound safety screen.
- `safety.screen_outbound(text) -> bool` (§24.4): run the deterministic tripwire on generated/outbound content; block + log a `safety_events` row (`direction="out"`) on a hit. Wire it before every outbound send (opener + normal replies).
- Consent record (§24.3): persist `opt_in_at` + number at signup; honor STOP via the per-number kill-switch.

- [ ] Test: a persona whose finetune job flips to `ready` sends exactly one templated opener via the (fake) sender, and the opener passes the outbound screen → commit `feat(messaging): reveal + proactive opener`.
- [ ] Test: `screen_outbound` blocks a crisis-laden generated string and logs an outbound `SafetyEvent`; a benign opener passes → commit `feat(safety): outbound content screening`.
- [ ] Test: a STOP from the friend silences the number (kill-switch) even for the opener path → commit.
